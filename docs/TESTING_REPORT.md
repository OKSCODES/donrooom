# Version 1.0 Testing Report

Release gates: clean dependency installation, environment-template validation, ESLint, production build, Vite development startup, SPA route responses, ZIP integrity, and clean-extraction rebuild.

Manual release checklist covers public routes; registration, login, verification, reset and logout; Guest, Property and Admin authorization; property/room/gallery CRUD; booking overlap rejection and adjacent-date acceptance; booking status changes; receipts and WhatsApp links; favorites, reviews, notifications, inquiries, filters, maps and SEO metadata; mobile layouts from 320px through 1920px; keyboard navigation, focus visibility, offline fallback, and installation prompt.

Live Firebase mutation and delivery testing must be performed in the selected Firebase staging project because credentials, email delivery, quotas, indexes, and deployed rules are environment-specific.
