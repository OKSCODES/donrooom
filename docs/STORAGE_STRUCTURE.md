# Firebase Storage Structure

| Path | Purpose | Write permission | Validation |
|---|---|---|---|
| `profile-images/{uid}/{file}` | User profile images | owning active user | JPEG, PNG or WebP; maximum 5 MB |
| `review-images/{uid}/...` | Guest review images | owning active user | JPEG, PNG or WebP; maximum 5 MB |
| `property-images/{uid}/...` | Property, room and gallery images | owning property account or admin | JPEG, PNG or WebP; maximum 5 MB |
| `site-assets/...` | Managed platform assets | admin only | JPEG, PNG or WebP; maximum 5 MB |

All other paths deny reads and writes. Upload components perform client-side validation and compression, but Storage rules are authoritative. File names must be generated by the application rather than trusted from user input. Backup procedures must preserve object paths and metadata.
